full_publish.py — primary listing tool

Location: E:\Book\Lister\full_publish.py — use this for bulk listing of pending books.

This is the correct tool when books need to be listed from scratch. It handles the full sequence: inventory item → offer → publish.

Sequence per book

  • Check BLOCKLIST — skip if ISBN is blocked
  • PUT inventory item — creates/updates with title, format, ISBN, qty=20, condition=NEW
  • DELETE any existing offers for this SKU
  • POST fresh offer WITH merchantLocationKey: "home1"
  • POST publish offer → live eBay listing
  • Update CSV: status=active, offer_id, sell_price, listed_at
  • Save CSV every 10 books

Why merchantLocationKey matters

CRITICAL: merchantLocationKey: "home1" MUST be in the offer POST payload. It is silently dropped on PUT. Without it eBay cannot resolve Item.Country and publish fails with error 25002. full_publish.py always POSTs fresh offers — never PUTs existing ones.

fix_listings.py — scoring, repricing, delisting

Location: E:\Book\Lister\fix_listings.py — manual run. Also synced to GitHub repo.

Phase 1 — scoring

  • Fetch eBay sold comps + active listings via Browse API for all 408 books
  • Compute undercut price → apply Amazon cap → compute profit
  • Filter: profit below $12 → skip
  • Check BLOCKLIST → skip blocked ISBNs
  • Determine to_list (pending above MIN_PROFIT) and to_delist (active below MIN_PROFIT)

Phase 2 — upsert (known bug)

The get_or_create_offer function uses an if False hack that always forces POST. This means it always tries to create a new offer even if one exists, resulting in "Offer entity already exists" error 25002 for books that already have offers. Books that are truly new (no existing offer) get listed correctly. Books with existing offers fail silently and stay pending.

  • AI description via Anthropic API (cached in CSV — only called if empty)
  • Image lookup via 5-source fallback chain
  • PUT inventory item → POST offer (always, due to if False hack) → publish_offer

Phase 3 — delist

  • DELETE offer for each book in to_delist list
  • Update CSV: status=delisted, delisted_at, delist_reason=unprofitable
