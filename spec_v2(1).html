lister.py v2 — correct listing sequence

This is the GitHub Actions listing script. Uses the same logic as full_publish.py locally.

  • Read opportunities from scan_opportunities.json
  • Skip BLOCKLIST ISBNs
  • PUT inventory item (title, format, ISBN, qty=20, condition=NEW)
  • DELETE any existing offers for this SKU
  • POST fresh offer WITH merchantLocationKey: "home1"
  • POST publish offer → live eBay listing
  • Update CSV: status=active, offer_id, sell_price, listed_at

Why merchantLocationKey is critical

MUST be in offer POST payload. Silently dropped on PUT. Without it eBay cannot resolve Item.Country and publish fails with error 25002.
